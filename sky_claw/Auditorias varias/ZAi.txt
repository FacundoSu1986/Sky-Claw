C. Concurrencia y estado
C1 [ALTO] — Provider lock serializa todas las llamadas LLM (router.py:82232)

python

async with self._provider_lock:
    response_data = await asyncio.wait_for(self._provider.chat(**chat_kwargs), timeout=120.0)
Contradice el principio del guardrail ("No asyncio.Lock wrapping provider I/O"). El lock es para hot-swap seguro, pero debería ser solo durante el swap, no durante 120s de HTTP. Snapshot bajo lock, llamada fuera:

python

async with self._provider_lock:
    provider_snapshot = self._provider
response_data = await asyncio.wait_for(provider_snapshot.chat(**chat_kwargs), timeout=120.0)
C2 [ALTO] — SupervisorAgent crea NetworkGateway propio (supervisor.py:100718)
AppContext ya construye uno y lo inyecta en el router, pero el supervisor crea otra instancia → dos caches DNS pinning separados, dos sets de reglas egress. Inyectar el gateway del AppContext.

C3 [MEDIO] — TaskGroup fail-fast para daemons (supervisor.py:100889)
Si un daemon cae, cancela todos. Los daemons deberían ser tolerantes a fallos transitorios. Patrón supervisor con reinicio individual y backoff.

C4 [MEDIO] — Truncado silencioso del contexto (router.py:82429)
messages = messages[-self._max_context:] descarta sin avisar al LLM. Para tool rounds con resultados críticos, causa "amnesia" del objetivo. Ya existe — usar siempre summarize.TokenBudgetManager.summarize_older_messages

C5 [MEDIO] — WAL pragma sin foreign_keys=ON (router.py:82027)
Las convenciones piden ambos. Si crece con FKs en el futuro, se romperá silenciosamente.chat_history

C6 [BAJO] — ContextManager abre conexión SQLite directa (context_manager.py:42254)
